# CityWork 影像资产管理平台 - 前端技术栈选型文档

## 1. 选型背景与核心原则

结合《前端架构设计文档》中关于“个人/团队空间资产管理”与“公共图库3D大屏展示”的核心业务需求，以及项目初期“暂时降级、后续迭代”的平滑演进策略。本次技术栈选型的核心原则为：**最简单（极低学习成本、开箱即用）且最健壮（生态成熟、类型安全、利于长期维护）**。

摒弃过度设计的花哨技术，采用目前前端工程化中最主流、最稳定的标准生态。

---

## 2. 核心框架与构建层 (最稳固的基础)

- **核心框架：Vue 3 (Composition API)**
  - **入选理由**：项目目录本身已锚定 Vue 生态。Vue 3 的 `<script setup>` 语法极大地简化了组件心智负担。相较于 React，Vue 3 的响应式系统更易于上手，且在处理复杂的表单和状态（如团队空间的人员管理、图片编辑表单）时代码更清晰。
- **开发语言：TypeScript**
  - **入选理由**：虽然增加了初期的类型定义成本，但对于一个涉及大量数据模型（`PictureVO`, `SpaceVO`, `UserVO` 等）的业务系统而言，TypeScript 是保证代码健壮性、重构安全性的**绝对必要条件**。
- **构建工具：Vite**
  - **入选理由**：Vue 官方首推构建工具，提供极速的冷启动和热更新，极大地提升开发体验，且开箱即用地支持 TypeScript 和 CSS 预处理器。

---

## 3. 路由与状态管理 (官方标准件)

- **路由管理：Vue Router 4**
  - **入选理由**：官方生态，无缝衔接 Vue 3。完美支持路由守卫（用于实现管理员入口鉴权、团队空间权限拦截）和动态路由（用于区分不同团队空间的 `/team/:spaceId`）。
- **状态管理：Pinia**
  - **入选理由**：Vue 官方推荐的新一代状态管理库。比 Vuex 更简单，抛弃了复杂的 `mutations`，完美支持 TypeScript。按照设计文档中的 `UserStore`, `SpaceStore`, `PictureStore` 划分领域状态，结构清晰且易于维护。

---

## 4. UI 组件与样式方案 (提效与统一)

- **核心组件库：Element Plus**
  - **入选理由**：国内 Vue 3 生态中最成熟、最健壮的 PC 端组件库。内置了完善的表格、表单、弹窗、上传组件（`<el-upload>`），能够直接覆盖本项目中复杂的“图片上传（后续平滑升级分片）”、“用户管理”、“审核列表”等后台交互场景，极大降低造轮子成本。
- **样式方案：Tailwind CSS + SCSS**
  - **入选理由**：
    - **Tailwind CSS**：原子化 CSS，用于快速搭建结构布局，减少随意命名的 class 堆砌，避免样式冲突，最适合快速迭代。
    - **SCSS**：作为 Tailwind 的补充，仅用于处理局部复杂的动画或 3D 容器的特殊层叠样式，两者结合达到最简开发体验。

---

## 5. 3D 可视化与图表 (平衡炫酷与简单)

- **图表引擎：ECharts**
  - **入选理由**：用于管理中枢的“数据分析”模块，渲染空间容量、标签分布等 2D 图表，国内绝对的行业标准。
- **3D 地图方案：ECharts 3D (ECharts-gl) 替代原生 Three.js**
  - **入选理由**：根据“最简单且健壮”的原则，直接手写 Three.js 来实现中国地图及数据映射的门槛极高、代码量大。**强烈推荐使用 ECharts 3D (ECharts-gl)**。它底层基于 WebGL，能以极简的配置实现 3D 中国地图、3D 柱状图（映射城市图片数量）和鼠标交互弹窗。既满足了“3D 视觉冲击力”，又将开发维护成本降低了 80%，极为健壮。

---

## 6. 网络与数据交互 (稳定可靠)

- **请求库：Axios**
  - **入选理由**：业界标准的 HTTP 客户端。通过简单的实例封装，即可实现全局 Token 注入（请求拦截）和统一的错误弹窗提示（响应拦截，结合 Element Plus 的 `ElMessage`）。对于现阶段的“常规直传”和未来的“分片上传”都有良好的扩展性支持。

---

## 7. 总结：技术栈架构图

```text
┌─────────────────────────────────────────────────────────────┐
│                       CityWork 业务层                       │
│  (个人空间 / 团队协同 / 公共3D图库 / 统一管理中枢)          │
├─────────────────┬─────────────────────┬─────────────────────┤
│   UI 组件库     │     3D 与可视化     │     状态与路由      │
│  Element Plus   │ ECharts / ECharts 3D│    Pinia / Router   │
├─────────────────┴─────────────────────┴─────────────────────┤
│                        核心框架                             │
│                  Vue 3 + TypeScript                         │
├─────────────────────────────────────────────────────────────┤
│                        基础设施                             │
│       Vite (构建) / Axios (网络) / Tailwind CSS (样式)      │
└─────────────────────────────────────────────────────────────┘
```

**方案优势说明**：
这套组合**没有任何实验性技术**，全部选用社区沉淀最久、文档最全的官方/主流库。特别是将 3D 地图的实现由原生 `Three.js` 降本增效为 `ECharts 3D`，在保证项目核心视觉价值的同时，极大降低了长期维护难度，是当前需求下**最简单且最健壮**的最佳实践。
